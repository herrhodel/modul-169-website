# LB 2: Praktische Prüfung

## Themen

- Dockerfiles erstellen können
- docker cli Befehle kennen: (`run`, `build`, `start`, `volume ..`,
  `container ..`, `network ..`, `inspect`, usw.)
- docker compose Befehle kennen (`up`, `down`, usw.)
- Mit docker compose Applikationen orchestrieren können (`docker-compose.yml`
  erstellen).
- Aus einem `docker-compose.yml` docker cli Befehle ableiten können.
- Aus `docker-compose.yml` ein Diagramm erstellen können.
- Diagramme lesen und verstehen können.
- Volumen und Bind Mounts erstellen und verwenden können.
- Netzwerke erstellen und verwenden können.
- Inspizieren von Images, Containers, Netzwerk und Volumes können.

## Umfeld

- Die Prüfung wird auf der Modul VM m169 geschrieben.
- Es wird eine Zip-Datei mit den Aufgaben zur Verfügung gestellt.
- Zusätzlich gibt es auf Papier die Anweisungen.
- Die Abgabe geschieht via Netzwerk Mount in der VM.

## Hilfe

- Es darf die [Modulwebseite](https://herrhodel.github.io/modul-169-website) als
  Spick verwendet werden.
- Es darf die [offizielle Referenz](https://docs.docker.com/reference/) auf
  docker.com verwendet werden.
  - **ACHTUNG:** der KI-Mode (Gordon) von dockerdocs **ist nicht erlaubt und
    führt zur Note 1**.

### Restriktionen

- Keine KI / AI
- Keine Suchmaschine
- Keine eigenen VS-Code Extensions installieren
